Data Management

Ko Ko Wai Yan

May 18, 2021

Topic - လူတွေ သေဆုံးသွားတဲ့အခါ သူတို့ရဲ့ Account တွေ Data တွေ ဘယ်လိုလုပ်မလဲ..?

လွန်ခဲ့တဲ့ ၅နှစ်လောက်က ဒီစကားကိုပြောရင် အရမ်းထူးစမ်းနေလိမ့်မယ် ဒီနေ့အခါမှာတော့ ဒါဟာ ထဲ့ပြီးစဥ်းစားသင့်တဲ့အရာတစ်ခု ဖြစ်လာပါပြီ။ များသောအားဖြင့်လူတွေရဲ့ နေ့စဥ်ဘဝတွေမှာ ဖုန်းတစ်လုံး၊ ဒါမှမဟုတ် Software တစ်ခုခုနဲ့ ထိတွေ့လာကျပါပြီ။ လူမှုရေး၊ ကျန်းမာရေး၊ စီးပွားရေး၊ ပညာရေး အစရှိတဲ့ နယ်ပယ်အစုံမှာ Software တွေကတဆင့် Server တွေ Cloud တွေ အစုံ ပေါ်မှာ Data တွေ စီးဆင်း ကျတယ်၊ သတင်းအချက်အလက်တွေ သိမ်းစည်းကျတယ်။ အဲ့တော့ ဒီလူတစ်ယောက် သေဆုံးသွားရင် သူ့ရဲ့ Data တွေ ဘယ်လိုလုပ်မလဲ?

Cloud Storage တစ်ခုက Data တွေ အများကြီး သိမ်းလို့ရတယ် အများကြီးဆိုပေမဲ့လဲ လိုအပ်သလို ပမာဏတိုးလို့ရတဲ့အတွက် ကန့်သတ်ချက်တော့ရှိတယ် ဒါကထားပါတော့လေ။ လူတွေရဲ့ နေ့စဥ်ဘဝမှာအသုံးပြုခဲ့တဲ့ Account တွေ Data တွေ Storage တွေ Data ထဲမှာ ပါပြီးသားဖြစ်တဲ့ Message တွေ Photo တွေ Video တွေ Voice တွေ..မြောက်များစွာသောအရာတွေ ဒါတွေက ကိုသေဆုံးသွားတဲ့အခါမှာ အကုန်လုံး Delect ဖြစ်မသွားဘူး။ သို့သော် ကိုမသေဆုံးခင်မှာ တစ်စုံတစ်ယောက်ကိုတော့ Power လွှဲခဲ့လို့ရတာရှိတယ်၊ သဘောကတော့ အဲ့ဒီလူက သေဆုံးသူရဲ Data တွေ ကို သူနောက်ဆုံးမှာခဲ့တဲ့ဆန္ဒ အတိုင်းလုပ်ပေးလို့ရတယ်ပေါ့။ ဒါက သိပ်တော့ ရုပ်တိမကျဘူးဗျ။ ဘာလို့လဲဆိုတော့ ဥပမာ Facebook လိုဟာမျိုးမှာက Memorialization ဆိုတာပါတယ်၊ ကိုယ်ခင်မင်ရတဲ့ Friend တစ်ယောက်ကို ထဲ့ထားလို့ရတယ် Account ကို အမှတ်တရ အဖြစ်ထားမယ်ဆိုလဲရတယ် ဖျက်မယ်ဆိုလဲရတယ် Message တွေကိုတော့ သူလဲတွေ့ခွင့်ရမှာမဟုတ်ဘူးပေါ့။ Google မှာကတော့ အချိန်ဘယ်လောက်အထိ ဒါကိုအသုံးမပြုရင် Delect လိုက်ပါ ဆိုတာမျိုး လုပ်ထားလို့ရတယ် Friend Mail တစ်ခုတော့ထဲ့လို့မရဘူး။ ကိုယ်ထားတဲ့အချိန်ပြည့်ဖို့နည်းရင် သူက အကြောင်းကြား​ပေးမယ် ၊ ဘာမှမလုပ်ပဲ ဒီတိုင်းထားရင် Account Delect သွားပြီပေါ့။ တစ်ချား Social Media တော်တော်များများမှာက အဲ့လိုမရှိဘူး။ ပြီးတော့ ကိုယ်သေမဲ့ရက်စွဲကိုလဲ ဘယ်သူမှအတိအကျမသိနိုင်ဘူး။ ဥပမာ ကိုယ်က Account တစ်ခုကို Phone No. နဲ့ ဖွင့်ပြီးသုံးတယ်ဆိုပါစို့ သူသေဆုံးသွားတဲ့အခါ အသုံးပြုခဲ့တဲ့ Account တွေ ထဲက Data တွေ Delect မသွားဘူး။ Mail မှာကလဲ Gmail မဟုတ်တဲ့ တချားသော Mail တွေအများကြီးရှိတယ် အဲ့ဒါတွေရော ဒီတိုင်းပဲရှိနေအုံးမှာပဲ။ အဲ့ဒါကိုစိတ်ထဲမှာ ဘဝင်မကျဘူးဖြစ်နေတာ၊ ခင်များတို့ကတော့ လက်ခံချင်မှလက်ခံပါ။ ကျွန်တော်အတွေးကတော့ သေဆုံးသွားရင် ဖျက်သင့်တာဖျက် ထားသင့်တာထား အဲ့လိုပဲရှိသင့်တယ်။ လူတွေက ရှင်သန်နေတဲ့အချိန်မှာ Data တွေ Manage ရှိပြီး သေးဆုံးသွားတဲ့အခါမှကျတော့ Manage မလုပ်ထားခဲ့နိုင်ဘူး( ရှင်သန်နေတဲ့အချိန်မှာ Manage မလုပ်တဲ့သူလဲရှိတယ် )။ အဲ့လိုပဲ သေဆုံးသူရဲ့ အသုံးမဝင်တဲ့ Data တွေ တကမ္ဘာလုံးမှာ ဘယ်လောက်တောင် ရှိနေလိုက်မလဲ။ တွက်ကြည့်မယ်ဆိုရင် အသုံးဝင်တဲ့ Storage ပမာဏ တခုရလိမ့်မယ် တန်ဖိုးရှိတယ်။ တကယ်လို့ ကမ္ဘာမြေပေါ်ကနေ တိပ်တိပ်လေးပျောက်ကွယ်သွားချင်တယ်ဆိုရင်တောင် အကြံအစီက အောင်မြင်မှာမဟုတ်ဘူး။ သူပိုင်တဲ့ Data တွေအတွက် သူဆိုတာ ရှိနေသေးတယ်။ 5G လို Network တွေ တွင်တွင်ကျယ်ကျယ် တကမ္ဘာလုံးမှာ အသုံးပြုလာရင် ပိုဆိုးမယ်။ အသုံးမဝင်တော့ဘူးဆိုရင်တော့ အဲ့ဒါကအမှိုက်ပဲ သေချာတာကတော့ ရှင်းရမယ်။ နောင်နှစ်ပေါင်းများစွာကြာခဲ့ရင် အဲ့ဒီအမှိုက်ပြသနာနဲ့ ရင်ဆိုင်ကောင်း ရင်ဆိုင်ရလိမ့်မယ်။ AI လို့ နည်းပညာတွေက ကူညီပေးနိုင်ရင်တော့ တစ်မျိုးပေါ့လေ။ ဒါတောင် Password မေ့သွားလို့ Account သစ်ဖွင့်လိုက်တဲ့လူတွေက ကမ္ဘာမှာ ၁မိနစ်မှာ အယောက် ၁၀၀ မကရှိတယ်။ အသုံးပြုသူတွေဘက်က လေးစားမှုရှိရင် Account ၁ခုတိုင်းစီမှာ အဓိပ္ပယ်ရှိနေမှာပါ။ ကို့ရဲ့ Data ကိုယ်တန်ဖိုးထားရမယ်၊ သေဆုံးသွားသည် အထိတိုင်အောင်ပေါ့။
